Vocabulary for the representation of data on the events that take place within the municipal agenda and their participants: Description back to ToC

Figure 1 presents the classes and properties of the vocabulary for the representation of data on the events that take place within the municipal agenda and their participants
The main concept of this vocabulary is [esagm:Event](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#Evento) for which the classes [schema:Event](http://schema.org/Event) and [schema:Place](http://schema.org/Place) of the schema.org vocabulary are reused. A Place can be a municipal building or center [esequip:Equipamiento](http://vocab.linkeddata.es/datosabiertos/def/urbanismo-infraestructuras/equipamiento-municipal) such as the town hall, a cultural center, etc. An event may have documentation [esagm:Documentacion](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#Documentacion) such as the Agenda or the Minutes of the meeting. A member of an event can have the role [esagm:rol](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#rol) of convener, holder or attendee. The events are composed by several [foaf:Agent](http://xmlns.com/foaf/0.1/Agent); it is worth mentioning that the set of agents includes people, organizations such as institutions or companies and groups. The positions of the participants in an event and its administrative units are represented through the classes [org:Role](https://www.w3.org/TR/vocab-org/#class-role), [org:Organization](https://www.w3.org/TR/vocab-org/#class-organization) and [org:Membership](https://www.w3.org/TR/vocab-org/#class-membership). An organization has an area of ​​competence that is a governmental administrative region represented by the class [dbo:GovernmentalAministrativeRegion](http://dbpedia.org/ontology/GovernmentalAdministrativeRegion) and it can be a district [esadm:Distrito](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#Distrito) or a municipality [esadm:Municipio](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#Municipio). Additionally, in this vocabulary several SKOS concept schemes have been used to represent the type of access, type of session, type of event or channel of an Event. In Figure 1 you can see several relationships with the SKOS concept class, however, the specific values ​​have not been specified because they are different depending on the taxonomy they represent. For example, the Session Type property [esagm:tipoSesion](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#tipoSesion) represents whether the session is Ordinary, Extraordinary or Extraordinary Urgent.

3.1 Examples of vocabulary usebackindex

In order to facilitate the understanding of some classes and properties of this vocabulary, some examples will be provided below.  First, for the representation of an event, an instance of the class [esagm:Evento](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#Evento) is created with the properties [schema:name](http://schema.org/name) (title of the event), [schema:startDate](https://schema.org/startDate) and [esagm:tipoEvento](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#tipoEvento) according to the taxonomy of event types that has been defined, in the example the event belongs to the type internal meeting. A unique identifier of the event has been generated from the combination of its title and its start date and time. The event is associated through the property [schema:location](https://schema.org/schemaLocation) with an instance of [schema:Place](http](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#ambitoEvento) the Madrid municipality that has been defined by the National Geographic Institute (ign) as the municipality-type resource with code 28079. In addition, an instance of the class [esagm:RolIntegranteEvento](index-en.html#http://vocab.ciudadesabiertas.es/def/sector-publico/agenda-municipal#RolIntegranteEvento) is defined, that allows to relate the event to each holder or assistant, in this case information is provided about the holder of the event for which a unique identifier has been generated from their names and surnames.
